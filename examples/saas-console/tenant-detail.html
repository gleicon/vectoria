This is the tenant's API key. Share it with the tenant so they can index products and run searches. It only resolves indexes within this tenant's namespace — other tenants' data is inaccessible with this key.

Getting started
  1. Create an index — use the form below. The index name identifies where products are stored (e.g. catalog).
  2. Load products — use the tenant's API key + the index name. Products go to POST /indexes/{index-name}/products.
  3. Search — click Search & Overrides on the index row. Searches hit POST /indexes/{index-name}/search with the tenant key.
  4. After a server restart — click reindex on the index row. BM25 is in-memory and must be rebuilt from stored products.

See API Guide for curl examples and bulk-load commands.

Indexes

Each index is an isolated product catalog. Use separate indexes for different catalogs (e.g. catalog, seasonal). Data is loaded to an index, not a tenant — the tenant key + index name in the URL determine where products go.
